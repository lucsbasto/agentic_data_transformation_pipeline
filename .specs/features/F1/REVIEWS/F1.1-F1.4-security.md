# Security review — F1.1 to F1.4

**Scope:** `3398dbc..382776c` (pre-hardening).
**Verdict:** LOW risk — solid foundations; two medium items addressed in commits `e24d1e4` + `277d79a`.

## Findings

| Sev | Summary | Location | Status |
|-----|---------|----------|--------|
| MED | `bind_context(**kwargs)` accepts arbitrary keys — a caller could accidentally bind `api_key=settings.anthropic_api_key` and the `SecretStr` would be serialized to logs since no scrub processor is configured | `src/pipeline/logging.py:64` | **Fixed in 277d79a** — `_redact_secrets` processor scrubs `SecretStr` values and key-name matches before `JSONRenderer`. |
| MED | `_update_status` accepts a freeform `status: str` with no validation against `BATCH_STATUSES`; the DB CHECK constraint catches it, but the resulting `sqlite3.IntegrityError` is not wrapped in `ManifestError`, leaking a raw traceback | `src/pipeline/state/manifest.py:217` | **Fixed in e24d1e4** — status is validated before the SQL runs; `sqlite3.IntegrityError` is caught and wrapped. |
| LOW | `pipeline_state_db` is a relative `Path` resolved against `project_root()` but there is no guard against `../../` segments — an env var like `PIPELINE_STATE_DB=../../tmp/evil.db` would create a DB outside the repo tree | `src/pipeline/settings.py:56` | **Fixed in 277d79a** — `_reject_path_traversal` field_validator denies `..` segments. |
| LOW | `ensure_schema` iterates `ALL_DDL` executing each statement via `cur.execute(stmt)` — safe today because `ALL_DDL` is a code constant, but if a future feature populates it from config/file, this becomes arbitrary SQL execution | `src/pipeline/state/manifest.py:99-100` | **Accepted risk** — `Final[tuple[str, ...]]` type pin documents the invariant; revisit only if DDL ever becomes data-driven. |

## Cleared areas

- **Secret handling:** `anthropic_api_key` uses `SecretStr`. `.env` is gitignored. `.env.example` has a placeholder only. Real keys never appeared in git history.
- **SQL injection:** All queries in `manifest.py` use `?` parameter binding exclusively. No string interpolation in SQL.
- **Path traversal:** `paths.py` resolves via `Path(__file__).resolve()` and walks upward to a marker — no user input involved.
- **Filesystem race conditions:** `mkdir(parents=True, exist_ok=True)` before `sqlite3.connect()` is the standard safe pattern. WAL journaling handles concurrent writers.
- **Dependency supply-chain:** All deps pinned with `>=` lower bounds; `uv.lock` present with exact hashes from PyPI. No vendored or private-registry packages. No known CVEs in the locked versions.
- **Log injection:** structlog's `JSONRenderer` encodes values as JSON strings, neutralizing newline/control-character injection.

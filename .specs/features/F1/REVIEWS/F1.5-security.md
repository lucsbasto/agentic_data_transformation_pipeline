# F1.5 Security Review — Bronze Ingest

**Verdict:** LOW risk. No exploitable vulnerabilities found; two theoretical hardening items noted.

## Findings

| Sev | Summary | Location | Fix |
|-----|---------|----------|-----|
| LOW | TOCTOU between `final_dir.exists()` and `rmtree` → `replace`: concurrent writers for the same `batch_id` could race. | `writer.py:64-66` | Single-writer is enforced by manifest `IN_PROGRESS`; add a file-lock if multi-process is added later. |
| LOW | No `n_rows` cap on `scan_source` / `write_bronze` — a maliciously large parquet could OOM the worker. | `reader.py:23`, `writer.py:62` | Accept `max_rows` kwarg; reject files above threshold before `.collect()`. |

## Priority Checklist — Cleared

1. **Path handling / atomic rename (writer.py):** `os.replace` via `Path.replace` is atomic on same FS. Temp dir lives under `bronze_root`. Crash cleanup of `.tmp-*` is present (line 54-55). `rmtree` before `replace` is a theoretical TOCTOU but single-writer is guaranteed by manifest locking — **cleared, LOW residual**.

2. **Symlink / directory-traversal:** `batch_id` is always hex-only (12 chars from `sha256().hexdigest()[:12]` in `batch.py:46`), never user-supplied free-text. No `/` or `..` can appear. `bronze_root` comes from `Settings` (Pydantic-validated), not user input — **cleared**.

3. **Parquet ingestion safety:** `scan_parquet` is lazy; no deserialization until `.collect()`. No known Polars CVEs as of 2026-04. Schema is strictly asserted before write (`assert_bronze_schema`). Missing: row-count cap — **cleared with LOW note** (see table).

4. **48-bit truncation collision:** Birthday bound is ~2^24 (16M) unique batches. Pipeline volume is orders of magnitude below this. An attacker cannot choose `batch_id` — it is derived from `sha256(content):mtime`, both server-side — **cleared, theoretical only**.

5. **Error messages leaking secrets:** Errors expose `source` path and `batch_id` (both non-secret operational data). No credentials, tokens, or PII in any error string. `mtime` is an integer timestamp — **cleared**.

6. **Fixture PII:** Phone numbers are synthetic patterns (`+55119XXXXXXX`), names are `Person 0`–`Person 4`. No real PII — **cleared**.

## Secrets & Dependencies

- **Secrets scan:** No hardcoded API keys, passwords, or tokens found in `src/pipeline/ingest/`.
- **Dependency audit:** `polars>=1.0`, `pyarrow>=16.0` — no known CRITICAL/HIGH CVEs. `pip-audit` not installed in env; recommend adding to CI.

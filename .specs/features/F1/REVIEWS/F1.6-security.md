## Cleared priorities

- **P1 — Click argument injection:** No shell expansion or `subprocess` use. Click coerces to `pathlib.Path`; no injection vector. `--source` requires `exists=True`. `--bronze-root` has the LOW above but no injection risk.
- **P2 — Logging secrets:** CLI logs only `source`, `bronze_root`, `batch_id`, `source_hash`, `source_mtime`, `duration_ms`, `error_type`. None carry secret values. `_redact_secrets` in `logging.py` covers any key matching `/secret|password|api[_-]?key|token|auth/i` plus all `SecretStr` instances. `anthropic_api_key` is `SecretStr` and never reaches a logger.
- **P3 — Manifest SIGKILL gap:** `reset_stale()` runs on every `ManifestDB.open()` (`manifest.py:91`), sweeping orphaned `IN_PROGRESS` rows to `FAILED`. Next run deletes and re-inserts (`cli/ingest.py:105-106`). Recovery is correct for single-process M1.
- **P4 — LLMClient stub secrets:** Stub raises `LLMError` with static strings only (`client.py:71-74, 79-80`). No `settings` fields in error text, no logging of `SecretStr`.
- **P5 — PipelineError → ClickException:** `cli/ingest.py:77` passes `str(exc)`. Exception messages contain only batch IDs, schema names, and file paths. `ConfigError` at `settings.py:81` wraps pydantic `ValidationError`, which does not include `SecretStr` values in its repr.
- **P6 — Test env leakage:** Both `test_cli_ingest_e2e.py` and `test_llm_client_stub.py` use `monkeypatch.delenv` for all 8 config keys before setting test values. `monkeypatch` auto-restores on teardown. No cross-test leakage risk.
